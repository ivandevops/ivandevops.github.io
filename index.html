关于数据库的基本知识
数据库分为关系型数据库和非关系型数据库（NoSQL）
关系型数据库适合处理结构化的数据，且数据的规模不能太大
非关系型数据库如mongodb，redis，memcache，简化数据库结构、避免影响性能的表链接，摒弃复杂的机构
关系型数据库是建立在关系模型上的，关系模型本质上就是存储若干数据的二维表
表的一行称为一条记录（record），表的一列称为字段（column）
关系型数据库的表之间需要建立一对一多对多的关系，这样才能够按照应用程序的逻辑来组织和存储数据。
关系是通过主键和外键进行维护的
主键是唯一区分不同记录的字段
外键是关联两张表的字段。外键会影响数据库性能
通过定义外键约束，关系型数据库可以保证无法插入无效的数据。eg：class表没有id=5的班级，那么student表也无法插入class_id为5的学生。
在关系型数据库中，如果有上万甚至上亿条记录，在查找记录的时候，想要获得非常快的速度，就需要使用索引。
约束（constraints）：创建表时规定约束或创建表后添加、删除、命名约束等,包括以下几种
NOT NULL, UNIQUE, PRIMARY KEY, FOREIGN KEY,CHECK,DEFAULT
UNIQUE 约束唯一标识数据库表中的每条记录。
UNIQUE 和 PRIMARY KEY 约束均为列或列集合提供了唯一性的保证。
PRIMARY KEY 拥有自动定义的 UNIQUE 约束。
请注意，每个表可以有多个 UNIQUE 约束，但是每个表只能有一个 PRIMARY KEY 约束。
FOREIGN KEY 一个表在创建的时候可以指定另外一个表中的某一个字段是他的某一个字段的外
CHECK 限制值的范围
DEFAULT 设置 默认值
NOT NULL 如果不向字段添加值，那么无法添加新的记录

使用github+hexo搭建个人博客

发表于 2020-01-08 | 分类于 blog开发
这个世界上最遥远的距离，就是知道和做到

很久以前就有搭建博客的想法
奈何行动力不足再加上能力不够就没有去做
希望可以通过博客记录来自己的学习和生活
以下就是我搭建博客的一些心得
希望可以帮助到更多的人

准备工作

环境配置

  • windows操作系统
  • 下载安装git （用于将在本地写好的博客推送到github page，需要配置本地主机与远程git的连接）
  • 下载安装node.js （npm会默认被安装）
  • 下载安装hexo，cmd管理员模式运行 npm install -g hexo （可能需要翻墙）

本地初始化博客内容

  • 选定一个目录，新建一个文件夹作为博客的根目录
  • 进入文件夹内，打开git bash，输入hexo init
  • 运行hexo s，此时访问本地4000端口，博客启动，安装成功

关联github

  • 在github上创建名字为XXX.github.io的项目，xxx是自己的github用户名
  • 打开本地的博客根目录的_config.yaml，设置配置文件如下
  • 运行npm install hexo-deployer-git –save
  • hexo g 生成本地静态文件
  • hexo d 将本地静态文件推送到git
  • 此时打开浏览器访问，http://ivandevops.github.io 能看到博客页面代表关联成功

配置博客的博文与主题（持续更新）

首先介绍一些重要的命令

  • hexo g （当你配置好博客后，用于生成要推送到github上的内容）
  • hexo d （把生成的博客推送到github）
  • hexo s （本地启动，用于调试）
  • hexo new “myfirstblog” （会在source/_posts文件夹内生成myfirstblog.md,这个文件夹就是放博客的文件夹）

已完成的博文配置

  • title 文章主题

  • date 创建日期

  • tags 标签

  • categories 分类

  • 文章中添加居中模块

已完成的主题配置

  • 博客标题、博客描述（签名）、作者、语言、时区

  • 博客主题的自定义以及配置
