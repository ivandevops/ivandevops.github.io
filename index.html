面试题-浏览器中输入一个url会发生什么

发表于 2020-01-15 | 分类于 面试准备
有何胜利而言，挺住就是一切

通过键盘输入网址： 操作系统会捕获到键盘的一个输入事件，会把这个url显示到界面上
输入回车: 操作系统也会捕获这个事件并反馈给浏览器
浏览器查找域名，并作dns解析查找域名的ip地址
如果浏览器已经缓存，那么就和这个ip的主机进行tcp的三次握手，进行一个tcp的连接
如果没有就会走一个dns的查询，会请求dns服务器拿到ip，再和这个主机进行三次握手在建立一个tcp的连接
握手完了之后，会把堆这个网址的请求，封装成一个数据包，基于http协议，发送到远端的服务器
一般的话远端是一个nginx服务器，nginx收到这个数据包后会进行解析，拿到这个url，交给具体处理该url的服务器
（一般这些服务器都会做负载均衡）服务器有缓存的话直接返回缓存（页面级），没有缓存的话，会访问数据库，或访问redis等，把需要的数据封装好，渲染完后发送给代理，代理再发送给前端，浏览器会把这个页面渲染出来。

关于数据库的基本知识
数据库分为关系型数据库和非关系型数据库（NoSQL）
关系型数据库适合处理结构化的数据，且数据的规模不能太大
非关系型数据库如mongodb，redis，memcache，简化数据库结构、避免影响性能的表链接，摒弃复杂的机构
关系型数据库是建立在关系模型上的，关系模型本质上就是存储若干数据的二维表
表的一行称为一条记录（record），表的一列称为字段（column）
关系型数据库的表之间需要建立一对一多对多的关系，这样才能够按照应用程序的逻辑来组织和存储数据。
关系是通过主键和外键进行维护的
主键是唯一区分不同记录的字段
外键是关联两张表的字段。外键会影响数据库性能
通过定义外键约束，关系型数据库可以保证无法插入无效的数据。eg：class表没有id=5的班级，那么student表也无法插入class_id为5的学生。
在关系型数据库中，如果有上万甚至上亿条记录，在查找记录的时候，想要获得非常快的速度，就需要使用索引。
约束（constraints）：创建表时规定约束或创建表后添加、删除、命名约束等,包括以下几种
NOT NULL, UNIQUE, PRIMARY KEY, FOREIGN KEY,CHECK,DEFAULT
UNIQUE 约束唯一标识数据库表中的每条记录。
UNIQUE 和 PRIMARY KEY 约束均为列或列集合提供了唯一性的保证。
PRIMARY KEY 拥有自动定义的 UNIQUE 约束。
请注意，每个表可以有多个 UNIQUE 约束，但是每个表只能有一个 PRIMARY KEY 约束。
FOREIGN KEY 一个表在创建的时候可以指定另外一个表中的某一个字段是他的某一个字段的外
CHECK 限制值的范围
DEFAULT 设置 默认值
NOT NULL 如果不向字段添加值，那么无法添加新的记录
